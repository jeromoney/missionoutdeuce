#MissionOut data validation specs
## email
 - An email should follow the RFC 5322 standard. The regex is (?:[a-z0-9!#$%&'*+/=?^_`{|}~-]+(?:\.[a-z0-9!#$%&'*+/=?^_`{|}~-]+)*|"(?:[\x01-\x08\x0b\x0c\x0e-\x1f\x21\x23-\x5b\x5d-\x7f]|\\[\x01-\x09\x0b\x0c\x0e-\x7f])*")@(?:(?:[a-z0-9](?:[a-z0-9-]*[a-z0-9])?\.)+[a-z0-9](?:[a-z0-9-]*[a-z0-9])?|\[(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?|[a-z0-9-]*[a-z0-9]:(?:[\x01-\x08\x0b\x0c\x0e-\x1f\x21-\x5a\x53-\x7f]|\\[\x01-\x09\x0b\x0c\x0e-\x7f])+)\])

 ## name
  - a name should following regex ^[\p{L}]+([ '\-\.][\p{L}]+)*$ The idea is to catch garbage, not define what a name is.

  ## phone number
  - a phone number should follow the E.164 standard, specifically the regex ^\\+[1-9]\\d{1,14}$ Flutter should use the phone_numbers_parser library to validate front end input. Entry should be assumed to be +1 United States but international.
