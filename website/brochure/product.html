Platform overview

The MissionOut system is built around dependable delivery and clear operational state.

Dispatch happens on the web, responder alerting is owned by native mobile code, and the API remains the system of record for incidents, responses, deliveries, and device health.

System model

Each component has one clear job in the alert chain.

Dispatcher web client Launches incidents and monitors acknowledgements from a focused live-operations interface.
API server Authenticates users and devices, stores incidents, and serves role-aware workflows.
Notification worker Fans alerts out to registered devices and records every delivery attempt.
Native alert handler Wakes the phone, starts the alarm path, and captures action even from the lock screen.
Escalation worker Applies retry, SMS, or voice policies when responders do not acknowledge quickly enough.
Device registry Maintains valid push tokens, platform metadata, and device health signals for targeting.

Operational flow

From incident creation to escalation, the full chain stays explicit and reviewable.

1

Dispatcher creates the incident

Priority, team, location, and notes are recorded in a structured form instead of a free-form text blast.

2

API writes the authoritative record

MissionOut stores incident state centrally so downstream workers and clients stay aligned.

3

Notification workers target valid devices

Delivery focuses on registered devices, which is more operationally useful than user-only targeting.

4

Native mobile code owns the urgent alert experience

The alarm path begins even if Flutter is backgrounded or cold-started, improving reliability under real conditions.

5

Responder intent syncs back immediately

Responding and unavailable actions become live operational data that dispatch can use right away.

6

Escalation policies keep the mission moving

Retries and fallback channels activate for non-responders, with every step preserved in the audit trail.

Role separation

MissionOut reduces mistakes by separating operational roles cleanly.

Responder

Receives mission alerts and submits immediate response intent from mobile devices.

Dispatcher

Creates incidents and monitors response status without inheriting unrelated admin permissions.

Team Admin

Manages one team's users, roles, device health, and team-scoped operational history.

Super Admin

Oversees all teams, system-wide controls, and cross-team operational visibility.
